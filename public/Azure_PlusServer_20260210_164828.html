Technical Analysis

Keycloak (PlusServer) is functionally superior for complex IAM requirements, while Entra External ID is still maturing.

In the 2025-2026 landscape, Microsoft Entra External ID is positioned as the modern successor to Azure AD B2C, but it currently suffers from significant functional regressions. Developer reports indicate that standard tasks—such as mapping a custom user attribute (e.g., memberId) to an ID token claim—are surprisingly difficult, often requiring Authentication Extensions (Azure Functions) or complex Schema Extension manipulations. The platform is described by some as 'fragmented,' with a steep learning curve for features that were native in B2C or are trivial in competitors.

PlusServer Keycloak, by contrast, delivers the full power of the mature Keycloak project. It excels in versatility: administrators can configure complex authentication flows, federate with arbitrary IdPs, and manipulate token claims via a robust GUI without writing external serverless functions. While PlusServer's managed nature imposes some infrastructure constraints (e.g., fixed node sizes, potential restrictions on custom JAR SPIs compared to bare metal), the underlying engine is far more capable and standards-compliant than Entra's current iteration.

For pure 'Hard Specs,' Keycloak's ability to handle complex authorization (AuthZ) and token transformation out-of-the-box earns it a higher score. Entra is only superior for teams strictly requiring 'no-ops' serverless scaling or deep integration with Microsoft 365/Intune signals.

Lock-in Analysis

Keycloak offers effectively zero vendor lock-in; Entra External ID is a high-lock-in walled garden.

  • PlusServer (Keycloak): The service utilizes the standard open-source Keycloak engine. You can perform a full Realm Export (JSON) at any time, which contains all users, roles, clients, and configuration. This file can be imported into any other Keycloak instance (self-hosted on AWS, another managed provider, or a local container) with near-zero friction.
  • Microsoft Entra External ID: User profiles, password hashes, and 'User Flow' configurations are proprietary to the Microsoft tenant. Migrating away requires using the Graph API to export raw user data (losing passwords) and rebuilding all authentication logic from scratch. There is no standard 'export' format compatible with other IDPs.

Summary: Azure Entra External ID is the clear winner for cost-efficiency for nearly all startups and small-to-mid-sized applications due to its generous free tier. PlusServer (Managed Keycloak) targets a different value proposition: data sovereignty (German data centers) and fixed-cost infrastructure for established enterprises.

  • Azure Entra External ID (SaaS): Operates on a consumption model. The first 50,000 Monthly Active Users (MAU) are free. Beyond that, pricing is roughly $0.03/MAU. This model is ideal for new projects because the bill is $0 until significant traction is achieved. However, costs scale linearly with success, potentially becoming expensive at millions of users.
  • PlusServer Managed Keycloak (PaaS/SaaS): Operates on a resource or service-fee model. You generally pay a monthly fee for the managed instance (clusters, high availability, backups) regardless of whether you have 1 user or 10,000. While 'Unlimited Users' is often a feature, the entry cost is high (typically starting in the hundreds of Euros/month for enterprise-grade managed IAM).
